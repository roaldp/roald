# Plan: Reduce Pulse Token Consumption

> **UPDATE 2026-03-14:** Root cause analysis below is outdated. Measurement shows system prompt + all tool schemas = only ~16K tokens (not 354K). The 96% majority is conversation content (tool results accumulating in context). `--allowedTools` does NOT filter loaded schemas. MCP server removal will save minimal tokens. See `.docs/plans/2026.03.14-pulse-token-observability.md` for the corrected analysis and data collection plan.

## High-Level Goal

Cut the recurring full_pulse token usage (currently ~400K tokens / $0.38 per run, every 30 min) by 50-70% without losing relevant context. The pulse must still answer: what's new, what's urgent, what can be delegated.

### Root Cause Analysis

Token breakdown for a single full_pulse (402K tokens):
- **354K cache_read** — System prompt, CLAUDE.md, MCP tool schemas for ALL connected/configured MCP servers
- **39K cache_creation** — Prompt + files read by Claude (mind.md, knowledge/index.md, mcp_tools.json)
- **8K output** — Claude's response + tool calls

The biggest cost drivers:
1. **MCP tool schemas for unauthenticated/unused servers** (~250K+ tokens) — `.context/mcp_tools.json` reveals 12 configured MCP servers, but only 2 are connected (Slack, Gmail). The other 10 (Asana, Atlassian, Box, Canva, Figma, HubSpot, Intercom, Linear, Notion, monday.com) show "Needs authentication" but their tool schemas are still likely loaded into the context window by the Claude CLI.
2. **Unverified assumption about `--allowedTools`** — The flag may only control which tools Claude can *call*, not which schemas are loaded. Must verify before relying on it.
3. **Full prompt repeated every 30 min** — No distinction between "first scan after startup" and "steady-state check." Prompt includes instructions for inactive sources (Calendar, Fireflies, Google Drive).
4. **No pre-filtering** — Claude is invoked even when there's nothing new.

## Important Technical Notes

### 1. Verify `--allowedTools` behavior first (Phase 0)

Before implementing tool filtering, we must verify that `--allowedTools` actually controls which tool schemas are loaded into the context. If it only controls which tools Claude can call (but still loads all schemas), the tool filtering approach saves zero tokens and we need a different strategy.

**Verification approach:** Run two Claude CLI invocations with `--output-format stream-json`:
- Minimal: `--allowedTools "Read"`
- Full: `--allowedTools "Read,mcp__claude_ai_Slack__*,mcp__claude_ai_Gmail__*,..."`

Compare `input_tokens` + `cache_creation_input_tokens` between both runs. If similar, `--allowedTools` does not filter schemas.

**Fallback strategy if `--allowedTools` doesn't filter:** Disconnect the 10 unauthenticated MCP servers from Claude CLI config (`claude mcp remove <server>`). This is the guaranteed way to stop their schemas from loading.

### 2. The real culprit: unauthenticated MCP servers

`.context/mcp_tools.json` shows 10 MCP servers in "Needs authentication" state (Asana, Atlassian, Box, Canva, Figma, HubSpot, Intercom, Linear, Notion, monday.com). These are configured globally in the Claude CLI, not by this project. Their tool schemas are loaded into every Claude invocation regardless of `--allowedTools`. Disconnecting these alone could save 200K+ tokens.

### 3. Share Slack state to avoid redundant pre-checks

`slack_loop` already polls Slack every 5 seconds. Instead of adding a separate pre-check that invokes Claude again, share `last_ts` state between `slack_loop` and `timer_loop` using an `asyncio.Event` or a shared variable. When `timer_loop` triggers a full pulse, it checks if `slack_loop` has seen new messages since last pulse — no additional Claude call needed.

### 4. Use config.yaml sources (already exists, currently unused)

`config.yaml` already defines `sources.slack`, `sources.gmail`, etc. as boolean toggles. `pulse.py` never reads them. Wire these up instead of parsing mind.md tables (fragile — written by Claude, formatting may vary).

### 5. Prompt cache efficiency

Keep the base prompt text stable across pulses. Append source fragments at the end so the prefix remains cache-friendly. Avoid randomizing or reordering sections between pulses.

### 6. mind.md pruning

The Inbox Zero Tracker table grows unboundedly. Add instructions to keep only the current pulse's entries. Older entries are already captured in knowledge/index.md. Note: prompt-based pruning is inherently unreliable (LLM may not follow). If it proves inconsistent, follow up with programmatic pruning in pulse.py (post-pulse Python step that truncates the table).

### 7. MCP server removal is a global, reversible operation

Removing unauthenticated MCP servers via `claude mcp remove` affects all Claude CLI usage, not just this project. Always snapshot the current config first (`claude mcp list > backup`). Servers can be re-added with `claude mcp add`. If the user later authenticates one of these services for another project, they will need to re-add it.

1. **Token reduction**: Full pulse uses < 250K tokens (from 402K) — verified via `scripts/token_report.py`
2. **Cost reduction**: Per-pulse cost drops below $0.20 (from $0.38)

5. **Logging**: Skipped pulses logged; token usage tracked per run
